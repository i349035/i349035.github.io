Make Your Own Blog with GitHub and Hexo

Posted on 2018-06-05 |

Requirements

  • Node.js
  • Git

Why GitHub Pages?

Free and Easy.

Create a repository

Head over to GitHub and create a new repository named username.github.io, where username is your username or organization name on GitHub.

More detailed information can be found in official website: https://pages.github.com

Choose a Framework

There are two popular blog frameworks: jekyll and hexo. Both of them are pretty, simple and Markdown support(Markdown is a lightweight and simple markup language, if you are not familiar with it, you can catch it from this Guide). Here I take Hexo as an example.

Installation

Setup

Once initialized, you can find the folder tree.

_config.yml

You can configure most settings here.

package.json

Application data.

scaffolds

You can create your own template here for creating a new post(article).

source

This is where you put your site’s content, Hexo ignores files or folders whose name are prefixed with _ - except _posts folder.

themes

Theme folder.

Writing

Create new post

Create a new post or a new page:

post is the default layout, you can create your own in scaffolds folder. You can edit your article in \MyBlog\source\_posts\My-Post.md.

Categories & Tags

In your post.md, you can edit categories & tags.

Format

More information can be found in official website: https://hexo.io/docs/tag-plugins.html

Here is only one tip: Use <!-- more --> to break the passage, the rest content can only be read by clicking “read more”.

Images

If you want to use the local images, follow steps:

Create post asset folder. You can create on your own or set it be auto.

Once you hexo new "Post", there is a folder created automatically called “Post”, you can just put images. (Of course, you can create the folder on your own.) Then insert image by this way in your post.md:

Server

Your website will run at http://localhost:4000 by default, it updates automatically, you can review your article in local.

Generating

Generating static files with Hexo.

After generating, you can find public folder created, all static files in this folder, you can copy to other place.
